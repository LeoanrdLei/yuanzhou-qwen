# 《听闻远方有你》写作实施计划 V1.0

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

# 用法: ./scripts/wordcount.sh [path]
# 统计 .md 文件正文字数（去除 markdown 标记的近似计数）
TARGET="${1:-manuscript}"
find "$TARGET" -name '*.md' -type f -print0 \
  | xargs -0 cat \
  | sed -E 's/[#*`>\[\]()_~`]+//g; s/^[[:space:]]*$//' \
  | tr -d '[:space:]' \
  | wc -m

### Task 0.2：写《00-风格守则》

**Files:**
- Create: `bible/00-风格守则.md`

这是全书写作的"宪法"。后续每章自检都对照此文。

- [ ] **Step 1: 起草《风格守则》**

内容要覆盖：

```markdown
# 《听闻远方有你》风格守则

## 一、语言基调
- 总调：金宇澄《繁花》南方调——口语高、细节多、场景增厚、含蓄克制
- 不写"心理分析"，写"动作、物件、对话"让读者自己感受
- 不抒情，不议论；情绪藏在物里、藏在话外
- 短句多于长句；不用排比；少用形容词，多用名词与动词

## 二、地域语言层次
- 黄山段：徽州茶馆味。可用本地词：毛峰、祁红、徽州话词汇（"晓得不""做啥""日头"），山间俚语
- 合肥段：江淮官话边缘地带。可用"中""能""可"等中原词
- 上海段：金宇澄式上海腔（不是上海话写实，是上海调）。"不响"句式可借鉴
- 杭州段：江浙调。可融合杭州本地与移民腔
- 旧金山段：英语/中式英语混杂，硅谷华人圈调。可用"H-1B、startup、return offer"等不翻译

## 三、视角守则
- 双视角：远舟（主声部）+ 若溪（次声部）
- 章内不切视角；视角切换以章为单位
- 远舟章用第三人称限制视角写远舟内心
- 若溪章用第三人称限制视角写若溪内心
- 配角内心永远不写，只从外部观察

## 四、时代细节守则
- 80 后同龄人共鸣，不解释——"QQ 点亮""人人网""非主流""魔兽世界""开心农场""校内"出现时不附加说明
- 但物件要写到具体型号：诺基亚 3310 / 摩托罗拉 V3 / iPhone 4 / 红米 Note
- 价格、工资、房价写到具体数字（参考各时点的真实价位）
- 流行语必查证当年是否出现——不能让 2003 年的人说 "yyds"

## 五、含蓄度守则
- 情感高潮处反而要降温——越想表达，越少字
- 重大事件不直接写"哭""崩溃"；写动作（攥拳、点烟、把窗关上）
- 对话句尾少用感叹号；少用问句堆叠
- 删除所有"他想""他感觉""他意识到"——改为外部呈现

## 六、章节字数与节奏
- 单章 3000-6000 字为主，关键场景章可到 8000 字
- 每卷开头有"时代锚定"段（1-2 段景物 + 1 个时代物件）
- 每卷结尾留一个"空镜头"（一个画面，不解释）

## 七、禁用清单
- ❌ "她明眸善睐""他剑眉星目"等套话
- ❌ "时光荏苒""岁月如梭"等过场套话
- ❌ "怦然心动""泪如雨下"等狗血词
- ❌ "命运""注定""缘分"等被言情污染的词
- ❌ 任何形式的"作者旁白评论"
```

- [ ] **Step 2: 自检——能否回答"如果我现在要写 2002 年雪天初见，遵循这份守则，会出什么样的句子"。如不能，补充细节**

- [ ] **Step 3: 提交**

```bash
git add bible/00-风格守则.md
git commit -m "docs(bible): 风格守则 V1"
```

## Phase 1：序章 + 卷一 少年得志（详细）

### Phase 1 通用章节写作循环（每章重复）

每章按以下 4 步走，每步是一个 commit：

```
分场景大纲 → 起稿 → 自检 → 终稿提交
```

**循环模板**（适用于所有章节）：

- [ ] **Step A: 分场景大纲**

把章节大纲展开为 3-5 个场景，每个场景标注：地点、人物、动作主线、字数预算、本章意象触发点。

写入 `docs/process-notes/<章号>-outline.md`。

提交：`git commit -m "outline: 第 X 章分场景"`

- [ ] **Step B: 起稿**

按场景大纲起稿全章。允许粗糙、允许语言不准、允许跳过卡住的句子（用 `[待补]` 标记）。

写入 `manuscript/<卷目录>/<章节文件>.md`。

提交：`git commit -m "draft: 第 X 章起稿"`

- [ ] **Step C: 自检**

对照 `bible/00-风格守则.md`，逐项检查：
- 禁用清单是否触雷
- 视角是否在章内切换
- 时代细节是否查证（流行词、价格、商品）
- 含蓄度是否过头（情绪是否被外化）
- 意象是否按 `bible/04-意象贯穿表.md` 出现

修订写入同一文件。提交：`git commit -m "revise: 第 X 章风格自检"`

- [ ] **Step D: 终稿确认**

通读一遍，删 `[待补]` 标记。运行字数统计：

```bash
./scripts/wordcount.sh manuscript/<卷目录>/<章节文件>.md
```

确认字数在预算范围内（±20%）。提交：`git commit -m "feat: 第 X 章终稿（XXXX 字）"`
