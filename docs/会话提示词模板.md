# 《听闻远方有你》分阶段会话提示词

> 用法：每开一次新会话，先粘贴【基础提示词】，再粘贴当次要做的【任务提示词】。
> 整本书 14-16 个月、80-100 章，每次会话只做一个 task，每个 task 一个 commit 周期。

---

## 一、基础提示词（每次会话开头必贴）

```
我在写一部 40 万字长篇小说《听闻远方有你》，是以我自己（80 后皖南人）为原型的时代史诗。
工作目录：/Users/leon/Documents/books/听闻远方有你

【必须先做的事——按顺序】
1. 读 docs/superpowers/specs/2026-06-20-novel-redesign-spec.md（设计稿，全书宪法，13 个核心决策都在里面）
2. 读 docs/superpowers/plans/2026-06-20-novel-writing-plan.md（实施计划，9 个 Phase 详尽到 step）
3. 读 bible/ 目录下所有已存在的文件（写作前置档案，是后续每章的参照）
4. 运行 git log --oneline 看一下已完成到哪一步

【行为规范】
- 你是主笔，我是 reviewer。你写、我改、你再改、commit。
- 严格遵守 bible/00-风格守则.md，特别是禁用清单。
- 每个 step 完成后停下来，告诉我"Step X 完成，[一句结果摘要]，是否继续？"
- 不准在没有完成上一个 step 时跳着做。
- 不准在 spec 之外擅自加情节、加人物、改设定。如有疑问先问我。
- 提交 commit 时用 plan 中给定的 message 格式（如 "docs(bible): ..."、"draft: 第 X 章起稿"、"feat: 第 X 章终稿（XXXX 字）"）。
- 文学项目，不准用 emoji、不准过度排版，正文用纯中文 markdown。

【今天的 Task】
（见下方"任务提示词"）
```

## 二、任务提示词（按 plan 阶段切换）

### A 型：Phase 0 准备类（写 bibles）

```
今天执行 Plan 中的 Task 0.X：[填具体 task 标题，例如 "Task 0.2：写《00-风格守则》"]

按 plan 中该 task 的 Step 1 / Step 2 / ... 顺序逐步执行。
正文起草后等我审阅，我可能让你重写或调整方向，重写后再 commit。
```

**填充示例**：
```
今天执行 Plan 中的 Task 0.3：写《01-人物档案》

按 plan 中该 task 的 Step 1 / Step 2 / ... 顺序逐步执行。
先列出 13 个角色清单确认，然后从林远舟开始一个一个写。每写完一个角色，给我看后我确认再写下一个。
```

---

### B 型：写章节大纲（每卷开头一次）

```
今天执行 Task X.0：卷 X 章节大纲展开（从 bible/06-章节大纲.md 的卷骨架展开到章详细）。

要求：
- 每章产出：章题、时点、地点、POV、200-400 字内容概要、关键场景列表、本章要触发的意象、字数预算
- 严格遵守 bible/05-双视角分配表.md 的 POV 归属
- 严格对齐 bible/02-时代年表.md 的时代细节
- 一章一章列，列完一章我点头你再列下一章
- 全部列完后追加到 bible/06-章节大纲.md（不是新建文件）
- 最后 commit："docs(bible): 卷 X 章节大纲展开"
```

---

### C 型：章节正文写作（核心循环，最常用）

```
今天执行 Task X.X：写 [章号 + 章题，例如 "卷一第 1 章 1998 年初秋"]。

执行顺序——四步，每步停下来等我确认：

Step A 分场景大纲
- 在 docs/process-notes/[章号]-outline.md 写本章 3-5 个场景，每个场景标：地点、人物、动作主线、字数预算、本章意象触发点
- 给我看，我确认后 commit："outline: 第 X 章分场景"

Step B 起稿
- 在 manuscript/[卷目录]/[章节文件].md 起草全章
- 允许粗糙、卡住的句子用 [待补] 标记
- 严格遵守 bible/00-风格守则.md
- 起完整章给我看，我可能让你重起或局部重写
- 我确认后 commit："draft: 第 X 章起稿"

Step C 风格自检
- 对照 bible/00-风格守则.md 逐项检查，特别是禁用清单
- 检查清单：①视角是否在章内切换 ②是否有"她想/他感觉/她意识到"等内心直白 ③时代细节查证 ④含蓄度是否过头 ⑤意象是否按 bible/04-意象贯穿表.md 出现
- 把发现的问题列出来，再修订
- 修订给我看，我确认后 commit："revise: 第 X 章风格自检"

Step D 终稿
- 通读删 [待补]，运行 ./scripts/wordcount.sh 看字数
- 字数预算 [填本章字数预算，从 bible/06-章节大纲.md 取] ± 20% 内
- commit："feat: 第 X 章终稿（XXXX 字）"

从 Step A 开始。
```

---

### D 型：整卷打通（每卷末一次）

```
今天执行 Task X.10/15：卷 X 整卷打通。

Step 1 全卷通读
- 把卷 X 所有章节文件按顺序通读
- 写《卷 X 读后笔记》到 docs/process-notes/卷X-读后笔记.md，包括：
  ①整体节奏是否符合 spec
  ②双视角是否突兀
  ③时代细节是否够浓
  ④哪章字数明显超/不足
  ⑤哪段读着无聊
- 给我看笔记，确认后 commit

Step 2 改第二稿
- 按笔记修订涉及的章节
- 每修订一章 commit 一次："revise(卷X): 第 X 章二稿"

Step 3 字数统计
- 运行 ./scripts/wordcount.sh manuscript/[卷目录]
- 对比 plan 中该卷字数预算，给我差异报告

Step 4 卷收尾日志
- 写 docs/process-notes/[日期]-phaseX-end.md，包括：本卷字数实际值、写作天数、需要回写到 bible 的设定调整、下卷心理预期
- commit："docs: Phase X 收尾日志"
```

---

### E 型：Phase 7-8 修订类

```
今天执行 Task 7.X / 8.X：[填具体 task 标题，例如 "Task 7.1：人物连贯性扫描"]。

严格按 plan 中该 task 的 step 执行。
扫描类任务用 grep 输出到 docs/process-notes/扫描-[关键词].md。
修订涉及的章节要逐章 commit，不要批量改后一次性 commit。
```

## 三、工作流建议

1. **保存"基础提示词"到笔记**——每次新会话第一条粘贴。本文件就是。
2. **按 plan 顺序走，不要跳**。下一次会话开始时让我跑 `git log --oneline` 我就知道走到哪了。
3. **一次会话只做一个 Task**，做完 commit、关掉，开新会话做下一个。这样每次上下文都干净，且你可以随时停下来思考。
4. **正文章节（Phase 1-6）我先全写，你 review**——如果你觉得我写的"不是你想要的腔调"，告诉我具体哪一段、为什么，我会调整再写。前 2-3 章是磨合期，磨出"你的声音"后我会越来越准。
5. **bibles 是基础设施，前 2-3 周专心做 Phase 0**——bibles 写得越细，后面章节写作越快、越稳。
6. **每周一次"周日小结"**：让我跑 `git log --since="7 days ago" --oneline` 给你一个周报，统计字数 + 已完成 task + 落后情况。
7. **如果某章你自己想动笔写**，告诉我"Step B 我自己来"，我跳过起稿、直接进 Step C 风格自检。这样保留你自己的语言指纹也行。
8. **若 spec 在写作过程中发现问题**（例如某个人物动作不通、某个时代细节查证后发现错），不要直接改章节，先回 spec / bible 改设定，再回章节，避免 spec 与稿子漂移。
9. **每完成一卷 git tag**：`git tag 卷一-V1` / `git tag 卷一-V2`，方便回滚与对照。
10. **alpha 读者**：终稿前找 1-2 位 80 后同龄人通读，他们最能挑出时代穿帮和共鸣失准。
